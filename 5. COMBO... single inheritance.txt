## 🟢 1. Basic Conceptual Questions

### ❓ What is this program about?

✔ It demonstrates **Single Inheritance** using `Animal` (superclass) and `Dog` (subclass), along with encapsulation and object usage.

---

### ❓ Why class is used?

✔ Class is a **blueprint** to create objects.
✔ It groups **data + methods together**.

👉 In this program:

* `Animal` class → stores animal data
* `Dog` class → extends Animal

---

### ❓ Why object is created?

✔ Object is needed to access class members.

👉 Example:

```java
Dog d = new Dog();
```

✔ Without object:

* No memory allocation
* No method execution

---

### ❓ Why Scanner is used?

✔ Scanner is used for **runtime user input**.

```java
Scanner sc = new Scanner(System.in);
```

---

## 🟢 2. Inheritance Questions

### ❓ Why inheritance is used?

✔ To achieve **code reusability** and avoid repetition.

---

### ❓ Why extends keyword is used?

✔ It connects child class to parent class.

```java
class Dog extends Animal
```

✔ Dog inherits:

* animalName
* animalType
* displayAnimal()

---

### ❓ What type of inheritance is this?

✔ Single Inheritance
(because Dog extends only Animal)

---

## 🟢 3. Encapsulation Questions

### ❓ Why private variables are used?

✔ To protect data (data hiding).

```java
private String animalName;
```

✔ Direct access not allowed outside class.

---

### ❓ Can subclass access private variables?

❌ No
✔ Only through public methods.

---

## 🟢 4. Methods Questions

### ❓ Why methods are used?

✔ To divide program into reusable blocks.

---

### ❓ Why method calling is needed?

✔ Methods only run when called.

```java
d.displayAnimal();
```

---

## 🟢 5. Constructor Questions

### ❓ Why constructor is not used here?

✔ Because we are manually setting values using methods.

---

### ❓ What happens if constructor is added?

✔ Object initialization becomes automatic.

---

## 🟢 6. Access Modifier Questions

### ❓ Why public methods?

✔ To allow access from other class.

---

### ❓ Why private variables?

✔ To restrict direct modification.

---

## 🟢 7. Static Keyword Questions

### ❓ Why static is used in main?

✔ JVM calls main method without object.

---

### ❓ What if static is removed from main?

❌ Program will NOT run
✔ JVM cannot find entry point

---

## 🟢 8. this Keyword Questions

### ❓ Why this is used?

```java

```

✔ It differentiates:

* class variable
* parameter variable

## 🟢 9. Polymorphism Questions

### ❓ Is polymorphism used here?

✔ Not directly
✔ No method overriding or overloading used

### ❓ Difference in this program context:

## 🟢 10. Tricky External Questions

### ❓ What if extends keyword is removed?

❌ No inheritance
✔ Dog cannot access Animal methods

---

### ❓ Can superclass object call subclass method?

❌ No
✔ Only subclass object can call both

---

### ❓ What if superclass method becomes private?

❌ It cannot be inherited

---

### ❓ What if object is not created?

❌ Program will not execute methods

---

### ❓ What happens if Scanner is not closed?

⚠ Memory leak risk (resource not released)

---

### ❓ Why String is immutable?

✔ Security + memory optimization + caching

---

## 🟢 11. JVM / JDK / JRE Questions

### ❓ What is JDK?

✔ Java Development Kit (compiler + tools)

---

### ❓ What is JRE?

✔ Java Runtime Environment (runs program)

---

### ❓ What is JVM?

✔ Converts bytecode into machine code

---

### ❓ What happens during execution?

1. Code compiled → .class file
2. JVM loads class
3. Heap memory allocates object
4. Methods executed
