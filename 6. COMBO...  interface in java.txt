# ⭐ 1. BASIC VIVA QUESTIONS

### 1. What is polymorphism?

Polymorphism means **one interface, many implementations**.

👉 In this program:

* `Shape s` behaves as `Circle`
* `Shape s` behaves as `Rectangle`

---

### 2. What is runtime polymorphism?

Runtime polymorphism occurs when **method call is decided at runtime**.

👉 Example:

```java
Shape s = new Circle();
s.calculateArea();
```

---

### 3. What is interface?

Interface is a blueprint of a class containing **abstract methods only**.

---

### 4. Why interface is used here?

To achieve:

* Abstraction
* Runtime polymorphism
* Loose coupling

---

### 5. What is class?

Class is a blueprint for objects.

---

### 6. What is object?

Object is an instance of a class stored in heap memory.

---

### 7. Why object is created?

To access class methods and variables.

---

### 8. Why Scanner is used?

To take **runtime input from user**.

---

### 9. Why main method is static?

So JVM can call it without creating object.

---

# ⭐ 2. OOP CONCEPT QUESTIONS

---

### 10. Where is polymorphism used?

```java
Shape s = new Circle();
Shape s = new Rectangle();
```

---

### 11. Is inheritance used here?

No direct class inheritance is used.
👉 But interface-based inheritance is used.

---

### 12. Why extends is not used?

Because interface is implemented using `implements`, not `extends`.

---

### 13. Why method overriding is used?

To provide **different implementation of same method**.

---

### 14. Difference between overloading and overriding?

| Overloading                            | Overriding            |
| -------------------------------------- | --------------------- |
| Same method name, different parameters | Same method signature |
| Compile time                           | Runtime               |

---

### 15. Why abstraction is used?

Because implementation is hidden from interface.

---
