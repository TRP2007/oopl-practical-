# 🟢 1. BASIC CONCEPTUAL QUESTIONS

### ❓ What is this program?

✔ It is a **menu-driven ATM simulation program** in Java using **exception handling**.

---

### ❓ What is a class?

✔ A class is a **blueprint of an object** which contains variables and methods.

---

### ❓ Why class is used here?

✔ Because ATM system needs:

* Data (balance)
* Behavior (deposit, withdraw, check balance)

So class groups them together.

---

### ❓ What is object?

✔ Object is an **instance of class used to access memory and methods**.

👉 In this program:
No explicit object of ATM is created because logic is inside `main()` (static context).

---

### ❓ Why object is not created here?

✔ Because `main()` is static, so execution starts without object.

---

### ❓ What is Scanner?

✔ Scanner is a **predefined class used to take input from user**.

---

### ❓ Why Scanner is used?

✔ To take:

* choice
* deposit amount
* withdrawal amount

---

### ❓ What is static?

✔ Static means **belongs to class, not object**.

---

### ❓ Why main is static?

✔ JVM calls main without creating object.

---

### ❓ What is method?

✔ Method is a **block of code that performs a specific task**.

Example:

* deposit logic
* withdraw logic

---

### ❓ Why methods are used?

✔ For:

* code reuse
* modularity
* readability

---

# 🟢 2. OOP CONCEPT QUESTIONS

---

### ❓ Is inheritance used here?

✔ ❌ No inheritance is used.

---

### ❓ Why inheritance is not used?

✔ Because ATM is a **single standalone system**, no parent-child relationship required.

---

### ❓ What is polymorphism?

✔ One action → many forms

❌ Not used explicitly here.

---

### ❓ What is abstraction?

✔ Hiding internal implementation

❌ Not used (no abstract class)

---

### ❓ What is encapsulation?

✔ Wrapping data + methods together in class.

✔ Used partially (balance variable + operations)

---

### ❓ Why extends keyword is not used?

✔ Because no parent class is defined.

---

# 🟢 3. EXCEPTION HANDLING QUESTIONS

---

### ❓ What is exception handling?

✔ Handling runtime errors using try-catch.

---

### ❓ Why exception handling is used?

✔ To prevent program crash during invalid transactions.

---

### ❓ Why ArithmeticException used?

✔ When withdrawal > balance.

---

### ❓ Why IllegalArgumentException used?

✔ When user enters negative/zero amount.

---

### ❓ What is finally block?

✔ Always executes after try-catch.

---

### ❓ Why finally is used?

✔ To show "Transaction Completed" always.

---

# 🟢 4. TRICKY EXTERNAL QUESTIONS

---

### ❓ What happens if Scanner is not closed?

✔ Memory leak may occur (resource not released).

---

### ❓ What happens if try-catch is removed?

✔ Program may crash on invalid input.

---

### ❓ What happens if balance is negative?

✔ Logical error in banking system.

---

### ❓ What if switch is replaced by if-else?

✔ Program will still work but less readable.

---

### ❓ Can we remove finally?

✔ Yes, but confirmation message may not always display.

---

### ❓ What happens if break is removed?

✔ Fall-through will occur (all cases execute).
